# 高考学习系统 - 快速部署命令清单

## 前置条件

- 域名 `www.ycx.com` 已解析到服务器 IP
- 服务器开放端口 80 和 443
- 你有 sudo 权限

## 一键部署

在终端执行以下命令（需要输入 sudo 密码）：

```bash
cd /home/ycx/learning_system/deploy
sudo ./auto-deploy.sh
```

脚本会自动完成所有部署步骤，包括：
1. 安装 nginx 和 certbot
2. 创建目录结构
3. 复制项目文件
4. 安装 Python 依赖
5. 配置 nginx 反向代理
6. 配置 systemd 服务
7. 申请 SSL 证书
8. 启动服务

## 手动部署（如果自动脚本失败）

### 1. 创建目录
```bash
sudo mkdir -p /opt/learning-system /var/lib/learning-system/{database,backup,progress,config} /var/log/learning-system /var/www/certbot
sudo chown -R www-data:www-data /var/lib/learning-system /var/log/learning-system
```

### 2. 复制文件
```bash
cd /home/ycx/learning_system
sudo rsync -av --delete \
    --exclude 'node_modules' \
    --exclude '.git' \
    --exclude '*.pyc' \
    --exclude '__pycache__' \
    --exclude '.env' \
    ./ /opt/learning-system/
sudo cp .env /opt/learning-system/.env
```

### 3. 启动服务器测试
```bash
cd /opt/learning-system
sudo -u www-data LEARNING_SYSTEM_DIR=/opt/learning-system LEARNING_SYSTEM_DATA=/var/lib/learning-system python3 unified_server.py &
```

### 4. 配置 nginx
```bash
sudo cp /opt/learning-system/deploy/nginx-yoursite.conf /etc/nginx/sites-available/learning-system
sudo ln -sf /etc/nginx/sites-available/learning-system /etc/nginx/sites-enabled/learning-system
sudo rm -f /etc/nginx/sites-enabled/default
sudo nginx -t
```

### 5. 配置 systemd 服务
```bash
sudo cp /opt/learning-system/deploy/learning-system.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable learning-system
```

### 6. 申请 SSL 证书
```bash
# 替换为你的邮箱
sudo certbot --nginx -d www.ycx.com -d ycx.com --non-interactive --agree-tos --email your-email@example.com
```

### 7. 启动服务
```bash
sudo systemctl start learning-system
sudo systemctl restart nginx
```

## 验证部署

```bash
# 检查服务状态
sudo systemctl status learning-system
sudo systemctl status nginx

# 测试访问
curl https://www.ycx.com/api/health

# 查看日志
sudo journalctl -u learning-system -f
```

## 常见问题

### 端口 80/443 被占用
```bash
sudo lsof -i :80
sudo lsof -i :443
```

### 证书申请失败
```bash
# 手动重新申请
sudo certbot --nginx -d www.ycx.com -d ycx.com
```

### 服务无法启动
```bash
# 查看详细错误
sudo journalctl -u learning-system -n 50 --no-pager

# 检查配置文件语法
python3 -m py_compile /opt/learning-system/unified_server.py
```
