# 高考学习系统 - HTTP 部署说明

## 部署完成！

系统已成功部署，使用 HTTP 协议（不使用 HTTPS）。

## 访问地址

- 本地访问：http://localhost
- 局域网访问：http://192.168.1.72
- 公网访问：http://116.8.1.195（需要在路由器配置端口转发）
- 域名访问（需配置 DNS）：http://www.ycx.com

### 端口转发配置（如需外部访问）

如果需要通过公网 IP 访问，在路由器上配置：
- 外部端口 80 → 内部 IP 192.168.1.72:80
- 外部端口 8080 → 内部 IP 192.168.1.72:8080

## 服务状态

```bash
# 查看学习系统状态
sudo systemctl status learning-system

# 查看 nginx 状态
sudo systemctl status nginx

# 查看日志
sudo journalctl -u learning-system -f
```

## 服务管理

```bash
# 重启学习系统
sudo systemctl restart learning-system

# 重启 nginx
sudo systemctl restart nginx

# 停止服务
sudo systemctl stop learning-system
sudo systemctl stop nginx
```

## 端口配置

- 80: nginx (HTTP)
- 8080: Python 后端服务

## 功能模块

- `/` - 学习系统主页
- `/tools` - 工具中心（时间跟踪、智能提醒、数据备份）
- `/mistake/` - 错题系统
- `/api/*` - API 接口
- `/account/*` - 账号服务

## 防火墙配置

如果需要从外部访问，确保开放端口：

```bash
# Ubuntu/Debian
sudo ufw allow 80/tcp
sudo ufw allow 8080/tcp

# CentOS/RHEL
sudo firewall-cmd --permanent --add-service=http
sudo firewall-cmd --permanent --add-port=8080/tcp
sudo firewall-cmd --reload
```

## DNS 配置（可选）

如果需要通过域名访问，在域名服务商添加：
- A 记录：`www.ycx.com` → `116.8.1.195`
- A 记录：`ycx.com` → `116.8.1.195`

## 如需升级到 HTTPS

```bash
# 1. 确保 DNS 已配置
# 2. 申请证书
sudo certbot --nginx -d www.ycx.com -d ycx.com --non-interactive --agree-tos --email your-email@example.com

# 3. 使用 deploy/nginx-yoursite.conf 替换 nginx 配置
```
