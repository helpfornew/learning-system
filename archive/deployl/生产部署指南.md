# 高考学习系统 - 生产部署指南

## 部署环境要求

- Ubuntu 22.04+ 或 Debian 12+
- 域名：www.ycx.com（已解析到服务器 IP）
- 开放端口：80 (HTTP), 443 (HTTPS)
- 用户权限：具有 sudo 权限

## 快速部署（推荐）

使用自动化部署脚本，脚本会自动处理所有依赖安装和配置：

```bash
cd /home/ycx/learning_system/deploy
sudo ./auto-deploy.sh
```

脚本会自动执行：
1. 安装系统依赖（nginx, certbot）
2. 创建目录结构
3. 复制项目文件
4. 安装 Python 依赖
5. 配置 nginx 反向代理
6. 配置 systemd 服务
7. 申请 SSL 证书

## 手动部署步骤

如果自动脚本不可用，可按以下步骤手动部署：

### 步骤 1：安装系统依赖

```bash
sudo apt update
sudo apt install -y nginx certbot python3-certbot-nginx
```

### 步骤 2：创建目录结构

```bash
# 安装目录
sudo mkdir -p /opt/learning-system
sudo mkdir -p /var/lib/learning-system/{database,backup,progress,config}
sudo mkdir -p /var/log/learning-system
sudo mkdir -p /var/www/certbot

# 复制项目文件
sudo rsync -av --delete \
    --exclude 'node_modules' \
    --exclude '.git' \
    --exclude '*.pyc' \
    --exclude '__pycache__' \
    --exclude '.env' \
    /home/ycx/learning_system/ /opt/learning-system/

# 复制环境配置
sudo cp /home/ycx/learning_system/.env /opt/learning-system/.env

# 设置权限
sudo chown -R www-data:www-data /var/lib/learning-system /var/log/learning-system
sudo chmod 755 /opt/learning-system
```

### 步骤 3：安装 Python 依赖

```bash
sudo pip3 install requests
```

### 步骤 4：配置 systemd 服务

创建服务文件：

```bash
sudo cp /opt/learning-system/deploy/learning-system.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable learning-system
```

### 步骤 5：配置 Nginx

```bash
# 复制 nginx 配置
sudo cp /opt/learning-system/deploy/nginx-yoursite.conf /etc/nginx/sites-available/learning-system
sudo ln -sf /etc/nginx/sites-available/learning-system /etc/nginx/sites-enabled/learning-system
sudo rm -f /etc/nginx/sites-enabled/default

# 测试 nginx 配置
sudo nginx -t
```

### 步骤 6：申请 SSL 证书

```bash
# 需要输入邮箱地址
sudo certbot --nginx -d www.ycx.com -d ycx.com --non-interactive --agree-tos --email your-email@example.com
```

### 步骤 7：启动服务

```bash
sudo systemctl start learning-system
sudo systemctl restart nginx

# 检查服务状态
sudo systemctl status learning-system
sudo systemctl status nginx
```

## 服务管理命令

```bash
# 查看服务状态
sudo systemctl status learning-system

# 重启服务
sudo systemctl restart learning-system

# 停止服务
sudo systemctl stop learning-system

# 查看日志
sudo journalctl -u learning-system -f

# 查看 nginx 日志
sudo tail -f /var/log/nginx/learning-system-error.log
```

## 目录结构

```
/opt/learning-system/          # 安装目录
├── unified_server.py          # 主服务器程序
├── .env                       # 环境变量配置
└── ...

/var/lib/learning-system/      # 数据目录
├── database/                  # 数据库文件
├── backup/                    # 备份文件
├── progress/                  # 学习进度
└── config/                    # 配置文件

/var/log/learning-system/      # 日志目录
└── server.log                 # 服务器日志
```

## 故障排查

### 服务无法启动

```bash
# 查看详细日志
sudo journalctl -u learning-system -n 50 --no-pager

# 检查端口占用
sudo lsof -i :8080
```

### Nginx 配置错误

```bash
# 测试配置
sudo nginx -t

# 查看错误日志
sudo tail -f /var/log/nginx/learning-system-error.log
```

### SSL 证书问题

```bash
# 重新申请证书
sudo certbot renew --dry-run
sudo certbot --nginx -d www.ycx.com -d ycx.com
```

## 更新部署

```bash
# 停止服务
sudo systemctl stop learning-system

# 更新代码
cd /opt/learning-system
sudo git pull  # 或者手动复制新文件

# 重启服务
sudo systemctl start learning-system
```
