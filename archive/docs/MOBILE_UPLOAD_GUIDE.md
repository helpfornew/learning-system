# 手机拍照上传错题指南

## 问题说明

**为什么外网 HTTP 访问无法使用摄像头？**

浏览器安全策略规定：`getUserMedia` API（摄像头）只能在以下环境使用：
- ✅ HTTPS 网站
- ✅ localhost / 127.0.0.1
- ❌ HTTP 外网访问（被浏览器阻止）

## 解决方案：手机拍照 + 上传页面

我们提供了一个**移动端优化的上传页面**，无需摄像头权限，直接调用手机相机或相册。

### 问题说明

### iPhone HEIC 格式问题

iPhone 默认使用 **HEIC/HEIF** 格式存储照片，可能在上传时遇到问题。

**解决方案：**
1. **更改相机格式**（推荐）：设置 → 相机 → 格式 → 选择「兼容性最佳」
2. **截图上传**：先截图再上传（截图自动转为 PNG）
3. **安装支持库**：服务器运行 `./install_heic_support.sh`

详细说明请查看 [HEIC_FORMAT_GUIDE.md](HEIC_FORMAT_GUIDE.md)

## 访问方式

```
http://your-server-ip:8080/mobile/
```

### 使用步骤

1. **手机浏览器访问**
   - 在手机浏览器输入：`http://你的服务器IP:8080/mobile/`
   - 或者电脑访问后生成二维码扫描

2. **选择科目**
   - 点击下拉菜单选择错题科目

3. **拍照/选择图片**
   - 点击上传区域
   - 选择「相机」直接拍照
   - 或选择「相册」从手机选择已有照片

4. **添加描述**（可选）
   - 简单描述这道错题的内容

5. **点击上传**
   - 等待上传完成
   - 成功后会显示提示

### 注意事项

1. **需要先登录**
   - 首次使用需要在电脑端登录
   - 手机端会自动使用相同的登录状态

2. **图片大小**
   - 单张图片最大支持 20MB
   - 上传前会自动压缩到合适的尺寸

3. **网络要求**
   - 手机和服务器需要在同一网络
   - 或者服务器有公网IP

### 常见问题

**Q: 提示"请先登录"？**
A: 先在电脑端登录系统，然后手机访问会自动继承登录状态。

**Q: 上传失败？**
A: 检查：1) 是否选择了科目 2) 图片是否超过20MB 3) 网络连接是否正常

**Q: 如何在电脑端查看上传的错题？**
A: 访问 `http://your-server-ip:8080/mistake/`，在错题本中查看。

**Q: 支持批量上传吗？**
A: 目前移动端上传页面暂不支持批量，建议电脑端使用批量上传功能。

### 技术说明

- 使用 `<input type="file" accept="image/*" capture="environment">` 调用相机
- 图片使用 Canvas 压缩，质量 85%，最大宽度 1920px
- 支持所有现代浏览器（Chrome, Safari, Edge, Firefox）

### 替代方案

如果手机上传页面无法满足需求，可以考虑：

1. **配置 HTTPS**（需要域名和 SSL 证书）
   ```bash
   # 生成自签名证书（仅测试）
   ./setup_https.sh
   ```

2. **使用内网穿透**（如 ngrok）
   ```bash
   ngrok http 8080
   # 会生成 https 链接，支持摄像头
   ```

3. **电脑连接手机**
   - 数据线连接手机到电脑
   - 手机选择文件传输模式
   - 电脑直接选择手机中的照片
