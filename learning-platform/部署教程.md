## 安装步骤

### 1. 下载项目

```bash
# 克隆或下载项目到本地
cd /home/user
# 假设项目文件夹名为 learning-hx
cd learning-hx
```

### 2. 安装 Python 依赖

```bash
# 安装 Flask 及相关依赖
pip3 install flask flask-jwt-extended werkzeug

# 或使用 requirements.txt（如果有的话）
pip3 install -r requirements.txt
```

**必需的包：**
- `flask` - Web 框架
- `flask-jwt-extended` - JWT 认证
- `werkzeug` - 密码加密

### 3. 初始化数据库

项目首次运行时会自动创建数据库，或者手动执行：

```bash
python3 -c "
import sqlite3
from werkzeug.security import generate_password_hash

conn = sqlite3.connect('database.db')
cursor = conn.cursor()

# 创建用户表
cursor.execute('''
    CREATE TABLE IF NOT EXISTS users (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        username TEXT UNIQUE NOT NULL,
        password_hash TEXT NOT NULL,
        is_admin INTEGER DEFAULT 0,
        created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
    )
''')

# 创建科目表
cursor.execute('''
    CREATE TABLE IF NOT EXISTS subjects (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        key TEXT UNIQUE NOT NULL,
        name TEXT NOT NULL,
        logo TEXT NOT NULL,
        color TEXT NOT NULL,
        description TEXT,
        sort_order INTEGER DEFAULT 0,
        is_active INTEGER DEFAULT 1,
        created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
        updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
    )
''')

# 创建章节表
cursor.execute('''
    CREATE TABLE IF NOT EXISTS chapters (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        subject_id INTEGER NOT NULL,
        key TEXT NOT NULL,
        title TEXT NOT NULL,
        description TEXT,
        content TEXT NOT NULL,
        sort_order INTEGER DEFAULT 0,
        is_active INTEGER DEFAULT 1,
        created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
        updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
        FOREIGN KEY (subject_id) REFERENCES subjects(id),
        UNIQUE(subject_id, key)
    )
''')

# 创建阅读记录表
cursor.execute('''
    CREATE TABLE IF NOT EXISTS reading_progress (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        user_id INTEGER NOT NULL,
        chapter_key TEXT NOT NULL,
        last_read_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
        read_count INTEGER DEFAULT 1,
        FOREIGN KEY (user_id) REFERENCES users(id),
        UNIQUE(user_id, chapter_key)
    )
''')

# 创建掌握记录表
cursor.execute('''
    CREATE TABLE IF NOT EXISTS mastered_chapters (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        user_id INTEGER NOT NULL,
        chapter_key TEXT NOT NULL,
        mastered_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
        FOREIGN KEY (user_id) REFERENCES users(id),
        UNIQUE(user_id, chapter_key)
    )
''')

# 创建涂鸦数据表
cursor.execute('''
    CREATE TABLE IF NOT EXISTS canvas_drawings (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        user_id INTEGER NOT NULL,
        chapter_key TEXT NOT NULL,
        drawing_data TEXT NOT NULL,
        updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
        FOREIGN KEY (user_id) REFERENCES users(id),
        UNIQUE(user_id, chapter_key)
    )
''')

conn.commit()
conn.close()
print('数据库初始化完成！')
"
```

## 配置说明

### 1. 配置文件 `config.js`

```javascript
// API 配置
const API_BASE = '/api';  // 相对路径，适用于前后端同域部署

// 其他配置
const CONFIG = {
    // API 基础地址
    apiBase: API_BASE,

    // Token 存储键名
    tokenKey: 'user_token',
    usernameKey: 'user_username',
    userIdKey: 'user_id',

    // 请求超时时间（毫秒）
    requestTimeout: 10000,

    // 同步状态显示时间（毫秒）
    syncStatusDuration: 2000
};
```

**修改 API_BASE 的情况：**
- 如果前后端分离部署，需要改为完整 URL，如 `http://api.example.com/api`

### 2. 后端配置 `app.py`

```python
# JWT 密钥（生产环境必须修改！）
app.config['JWT_SECRET_KEY'] = 'your-secret-key-here'  # 改为随机字符串

# Token 过期时间
app.config['JWT_ACCESS_TOKEN_EXPIRES'] = timedelta(days=7)

# 数据库文件路径
DATABASE = 'database.db'
```

**安全建议：**
- 使用环境变量存储 JWT_SECRET_KEY
- 定期更换密钥
- 设置合理的 Token 过期时间

## 启动服务

### 开发环境启动

```bash
# 进入项目目录
cd /path/to/learning-hx

# 启动 Flask 服务
python3 app.py
```

服务默认运行在 `http://0.0.0.0:5000`

### 生产环境部署（使用 Gunicorn）

```bash
# 安装 Gunicorn
pip3 install gunicorn

# 启动服务（4个工作进程）
gunicorn -w 4 -b 0.0.0.0:5000 app:app
```

### 后台运行（Linux）

```bash
# 使用 nohup
nohup python3 app.py > app.log 2>&1 &

# 或使用 systemd（推荐）
```

### Systemd 服务配置

创建文件 `/etc/systemd/system/learning-platform.service`：

```ini
[Unit]
Description=Learning Platform
After=network.target

[Service]
Type=simple
User=www-data
WorkingDirectory=/path/to/learning-hx
ExecStart=/usr/bin/python3 /path/to/learning-hx/app.py
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

启动服务：

```bash
sudo systemctl enable learning-platform
sudo systemctl start learning-platform
sudo systemctl status learning-platform
```

## 管理员设置

### 1. 创建管理员账号

```bash
python3 -c "
import sqlite3
from werkzeug.security import generate_password_hash

conn = sqlite3.connect('database.db')
cursor = conn.cursor()

# 创建管理员账号
password_hash = generate_password_hash('your-admin-password')
cursor.execute('''
    INSERT OR REPLACE INTO users (id, username, password_hash, is_admin)
    VALUES (1, 'admin', ?, 1)
''', (password_hash,))

conn.commit()
conn.close()
print('管理员账号创建成功！')

print('密码: your-admin-password')
"
```

## 常见问题

### Q1: 端口被占用

**错误信息：**
```
Address already in use
Port 5000 is in use by another program
```

**解决方法：**

```bash
# 查找占用 5000 端口的进程
lsof -i :5000

# 或 Linux
fuser -k 5000/tcp

# 然后重启服务
```

### Q2: 数据库权限错误

**错误信息：**
```
sqlite3.OperationalError: attempt to write a readonly database
```

**解决方法：**

```bash
# 修改数据库文件权限
chmod 666 database.db

# 或更改所有者
chown www-data:www-data database.db
```

### Q3: 跨域问题（CORS）

**症状：** 前端无法调用 API

**解决方法：**

后端 `app.py` 已配置 CORS，如需修改：

```python
@app.after_request
def after_request(response):
    response.headers.add('Access-Control-Allow-Origin', 'http://your-domain.com')
    response.headers.add('Access-Control-Allow-Headers', 'Content-Type,Authorization')
    response.headers.add('Access-Control-Allow-Methods', 'GET,PUT,POST,DELETE,OPTIONS')
    return response
```

### Q4: 静态文件 404

**症状：** CSS/JS 文件加载失败

**解决方法：**

确保 `app.py` 中静态文件夹配置正确：

```python
app = Flask(__name__, static_folder='.', static_url_path='')
```

### Q5: 修改 JWT 密钥后登录失败

**症状：** 已登录用户提示重新登录

**说明：** 修改 JWT_SECRET_KEY 后，所有已颁发的 Token 会失效，用户需要重新登录。这是正常行为。

## 生产环境建议

### 1. 使用 Nginx 反向代理

```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://127.0.0.1:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

### 2. 启用 HTTPS

使用 Let's Encrypt 免费证书：

```bash
sudo certbot --nginx -d your-domain.com
```

### 3. 数据库备份

定期备份数据库：

```bash
# 备份脚本
#!/bin/bash
DATE=$(date +%Y%m%d_%H%M%S)
cp /path/to/database.db /backup/database_${DATE}.db

# 保留最近 7 天的备份
find /backup -name "database_*.db" -mtime +7 -delete
```

### 4. 日志管理

```bash
# 查看日志
tail -f app.log

# 日志轮转配置 /etc/logrotate.d/learning-platform
/path/to/learning-hx/app.log {
    daily
    rotate 7
    compress
    delaycompress
    missingok
    notifempty
    create 644 user user
}
```

### 5. 安全建议

- ✅ 修改默认的 JWT_SECRET_KEY
- ✅ 使用强密码（至少 12 位，包含大小写和特殊字符）
- ✅ 定期更新依赖包 `pip3 install --upgrade flask`
- ✅ 限制管理后台访问 IP（可通过 Nginx 配置）
- ✅ 启用防火墙，只开放必要端口
