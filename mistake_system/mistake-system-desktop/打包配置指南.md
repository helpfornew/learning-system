# Electron 客户端打包配置指南

## 概述

客户端已经配置好 flexible API 地址系统，支持多种配置方式。

## 配置方式

### 方式 1：使用配置文件（推荐）

在打包前，将 `electron/config.prod.json` 复制为 `electron/config.json`：

```bash
cd /home/ycx/learning_system/mistake_system/mistake-system-desktop

# 复制生产环境配置
cp electron/config.prod.json electron/config.json

# 或者手动创建配置文件
cat > electron/config.json << EOF
{
  "server": {
    "url": "http://116.8.1.195",
    "enableCustomServer": true,
    "allowInsecureConnection": true
  }
}
EOF
```

### 方式 2：使用环境变量

创建 `.env.production` 文件：

```bash
cat > .env.production << EOF
# 生产环境 API 配置
VITE_API_BASE_URL=http://116.8.1.195
VITE_ACCOUNT_API_URL=http://116.8.1.195/account
VITE_API_TIMEOUT=30000
VITE_API_RETRY_ATTEMPTS=3
VITE_DEBUG_MODE=false
EOF
```

### 方式 3：用户运行时配置

客户端内置了服务器配置界面，用户可以在设置页面修改服务器地址。

配置文件位置：
- Linux: `~/.config/mistake-system-desktop/config.json`
- Windows: `%APPDATA%/mistake-system-desktop/config.json`
- macOS: `~/Library/Application Support/mistake-system-desktop/config.json`

## 打包步骤

### 1. 配置服务器地址

```bash
cd ~/learning_system/mistake_system/mistake-system-desktop

# 使用生产环境配置
cp electron/config.prod.json electron/config.json
```

### 2. 构建并打包

```bash
# Linux 版本（deb 包）
npm run build:linux

# Windows 版本
npm run build:win

# Mac 版本
npm run build:mac
```

### 3. 输出位置

打包完成后，安装包位于 `release/` 目录：
- `release/*.deb` - Linux 安装包
- `release/*.exe` - Windows 安装包
- `release/*.dmg` - Mac 安装包

## API 配置优先级

1. **用户配置** - 用户在设置页面配置的地址
2. **electron/config.json** - 打包时的配置文件
3. **默认配置** - `http://your-domain.com`

## 部署到服务器后的更新

如果服务器地址变更，有三种方式更新：

### 方法 1：更新配置文件（批量部署）

修改 `electron/config.json` 后重新打包：

```bash
cat > electron/config.json << EOF
{
  "server": {
    "url": "http://新服务器IP",
    "enableCustomServer": true,
    "allowInsecureConnection": true
  }
}
EOF

npm run build:linux
```

### 方法 2：用户自行配置

用户可以在客户端的设置页面修改服务器地址。

### 方法 3：使用外部脚本配置

创建启动脚本设置全局变量：

```bash
#!/bin/bash
export SERVER_URL="http://116.8.1.195"
./mistake-system-desktop
```

## 开发环境调试

```bash
# 启动开发服务器（Vite + Electron）
npm run dev:electron

# 开发环境会自动使用当前页面 hostname 作为 API 地址
# 例如访问 http://192.168.1.100:5173，API 会请求 http://192.168.1.100:8080
```

## 注意事项

1. **HTTP vs HTTPS**: 当前配置允许 HTTP 连接（`allowInsecureConnection: true`），适合内网部署
2. **CORS**: 后端服务器需要配置允许跨域访问
3. **防火墙**: 确保客户端能访问服务器 8080 端口
